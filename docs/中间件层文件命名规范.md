#### 中间件层文件命名规范

```
adapter_chip、adapter_core 两个目录下创建新文件的规范：

1.文件名称必须是英文26个字母或下划线"_"、下标"."组成
2.文件名称必须是ai前缀开头
3.文件含义使用下划线"_"进行连接
4.文件名称必须是可以清晰表达含义，需要说清楚为何这样命名、理由
5.文件名称不适宜太长，以便有更好的可观性
```

.

│   ├── bl602
